## Overview

`TableCalendar` is the main widget of the package. It's a highly customizable, feature-packed Flutter calendar with gestures, animations and multiple formats.

```dart
TableCalendar(
  firstDay: DateTime.utc(2010, 10, 16),
  lastDay: DateTime.utc(2030, 3, 14),
  focusedDay: DateTime.now(),
)
```

## Constructor

```dart
TableCalendar<T>({
  Key? key,
  required DateTime focusedDay,
  required DateTime firstDay,
  required DateTime lastDay,
  DateTime? currentDay,
  dynamic locale,
  DateTime? rangeStartDay,
  DateTime? rangeEndDay,
  // ... additional properties
})
```

## Properties

### Core Date Properties

<ParamField path="focusedDay" type="DateTime" required>
  DateTime that determines which days are currently visible and focused.
  
  This property controls which calendar page is displayed. Updating this value will change the visible month/week.
</ParamField>

<ParamField path="firstDay" type="DateTime" required>
  The first active day of `TableCalendar`. Blocks swiping to days before it.
  
  Days before this date will use `disabledStyle` and trigger `onDisabledDayTapped` callback.
</ParamField>

<ParamField path="lastDay" type="DateTime" required>
  The last active day of `TableCalendar`. Blocks swiping to days after it.
  
  Days after this date will use `disabledStyle` and trigger `onDisabledDayTapped` callback.
</ParamField>

<ParamField path="currentDay" type="DateTime?">
  DateTime that will be treated as today. Defaults to `DateTime.now()`.
  
  Overriding this property might be useful for testing.
</ParamField>

### Range Selection Properties

<ParamField path="rangeStartDay" type="DateTime?">
  The start of the selected day range.
  
  Used when `rangeSelectionMode` is enabled to mark the beginning of a date range.
</ParamField>

<ParamField path="rangeEndDay" type="DateTime?">
  The end of the selected day range.
  
  Used when `rangeSelectionMode` is enabled to mark the end of a date range.
</ParamField>

<ParamField path="rangeSelectionMode" type="RangeSelectionMode" default="RangeSelectionMode.toggledOff">
  Current mode of range selection.
  
  - `RangeSelectionMode.disabled` - range selection is always off
  - `RangeSelectionMode.toggledOff` - range selection is currently off, can be toggled by longpressing a day cell
  - `RangeSelectionMode.toggledOn` - range selection is currently on, can be toggled by longpressing a day cell
  - `RangeSelectionMode.enforced` - range selection is always on
</ParamField>

### Calendar Format & Layout

<ParamField path="calendarFormat" type="CalendarFormat" default="CalendarFormat.month">
  Specifies `TableCalendar`'s current format.
  
  Available formats:
  - `CalendarFormat.month` - shows a full month view
  - `CalendarFormat.twoWeeks` - shows two weeks
  - `CalendarFormat.week` - shows a single week
</ParamField>

<ParamField path="availableCalendarFormats" type="Map<CalendarFormat, String>" default="{ CalendarFormat.month: 'Month', CalendarFormat.twoWeeks: '2 weeks', CalendarFormat.week: 'Week' }">
  Map of `CalendarFormat`s and `String` names associated with them. Those `CalendarFormat`s will be used by internal logic to manage displayed format.
  
  To ensure proper vertical swipe behavior, `CalendarFormat`s should be in descending order (i.e. from biggest to smallest).
  
  ```dart
  availableCalendarFormats: const {
    CalendarFormat.month: 'Month',
    CalendarFormat.week: 'Week',
  }
  ```
</ParamField>

<ParamField path="startingDayOfWeek" type="StartingDayOfWeek" default="StartingDayOfWeek.sunday">
  `TableCalendar` will start weeks with provided day.
  
  - Use `StartingDayOfWeek.monday` for Monday - Sunday week format
  - Use `StartingDayOfWeek.sunday` for Sunday - Saturday week format
</ParamField>

<ParamField path="weekendDays" type="List<int>" default="[DateTime.saturday, DateTime.sunday]">
  List of days treated as weekend days.
  
  Use built-in `DateTime` weekday constants (e.g. `DateTime.monday`) instead of `int` literals (e.g. `1`).
  
  ```dart
  weekendDays: [DateTime.friday, DateTime.saturday]
  ```
</ParamField>

### Visibility & Behavior

<ParamField path="headerVisible" type="bool" default="true">
  Determines the visibility of calendar header.
  
  Set to `false` to hide the header containing month/year title and navigation chevrons.
</ParamField>

<ParamField path="daysOfWeekVisible" type="bool" default="true">
  Determines the visibility of the row of days of the week.
  
  Set to `false` to hide the row showing weekday names (Mon, Tue, Wed, etc.).
</ParamField>

<ParamField path="weekNumbersVisible" type="bool" default="false">
  Whether to display week numbers on calendar.
  
  When `true`, displays ISO week numbers in a column on the left side of the calendar.
</ParamField>

<ParamField path="pageJumpingEnabled" type="bool" default="false">
  When set to true, tapping on an outside day in `CalendarFormat.month` format will jump to the calendar page of the tapped month.
  
  By default, tapping outside days updates the focused day without changing the calendar page.
</ParamField>

<ParamField path="pageAnimationEnabled" type="bool" default="true">
  When set to true, updating the `focusedDay` will display a scrolling animation if the currently visible calendar page is changed.
</ParamField>

<ParamField path="sixWeekMonthsEnforced" type="bool" default="false">
  When set to true, `CalendarFormat.month` will always display six weeks, even if the content would fit in less.
  
  This ensures consistent calendar height across months.
</ParamField>

<ParamField path="shouldFillViewport" type="bool" default="false">
  When set to true, `TableCalendar` will fill available height.
  
  Useful when you want the calendar to expand to fill its parent container.
</ParamField>

### Dimensions

<ParamField path="rowHeight" type="double" default="52.0">
  Used for setting the height of `TableCalendar`'s rows.
  
  Each week row will have this height in pixels.
</ParamField>

<ParamField path="daysOfWeekHeight" type="double" default="16.0">
  Used for setting the height of `TableCalendar`'s days of week row.
  
  The row containing weekday names (Mon, Tue, Wed, etc.) will have this height in pixels.
</ParamField>

### Animations

<ParamField path="formatAnimationDuration" type="Duration" default="Duration(milliseconds: 200)">
  Specifies the duration of size animation that takes place whenever `calendarFormat` is changed.
</ParamField>

<ParamField path="formatAnimationCurve" type="Curve" default="Curves.linear">
  Specifies the curve of size animation that takes place whenever `calendarFormat` is changed.
</ParamField>

<ParamField path="pageAnimationDuration" type="Duration" default="Duration(milliseconds: 300)">
  Specifies the duration of scrolling animation that takes place whenever the visible calendar page is changed.

<ParamField path="pageAnimationCurve" type="Curve" default="Curves.easeOut">
  Specifies the curve of scrolling animation that takes place whenever the visible calendar page is changed.

### Gestures

<ParamField path="availableGestures" type="AvailableGestures" default="AvailableGestures.all">
  Specifies swipe gestures available to `TableCalendar`.
  
  - `AvailableGestures.all` - all gestures enabled
  - `AvailableGestures.horizontalSwipe` - only horizontal swiping enabled
  - `AvailableGestures.verticalSwipe` - only vertical swiping enabled
  - `AvailableGestures.none` - calendar will only be interactive via buttons
</ParamField>

<ParamField path="simpleSwipeConfig" type="SimpleSwipeConfig" default="SimpleSwipeConfig(verticalThreshold: 25.0, swipeDetectionBehavior: SwipeDetectionBehavior.continuousDistinct)">
  Configuration for vertical swipe detector.
  
  This is provided by the `simple_gesture_detector` package and controls swipe sensitivity and behavior.
</ParamField>

<ParamField path="dayHitTestBehavior" type="HitTestBehavior" default="HitTestBehavior.opaque">
  `HitTestBehavior` for every day cell inside `TableCalendar`.
  
  Controls how the day cells respond to hit testing during gestures.
</ParamField>

### Localization

<ParamField path="locale" type="dynamic">
  Locale to format `TableCalendar` dates with, for example: `'en_US'`.
  
  If nothing is provided, a default locale will be used. This affects month names, weekday names, and other date formatting.
  
  ```dart
  TableCalendar(
    locale: 'pl_PL',
    // ...
  )
  ```
</ParamField>

### Event Loading

<ParamField path="eventLoader" type="List<T> Function(DateTime day)?">
  Function that assigns a list of events to a specified day.
  
  This function is called for each visible day to load events. The generic type `T` represents your event type.
  
  ```dart
  eventLoader: (day) {
    return eventsMap[day] ?? [];
  }
  ```
</ParamField>

<ParamField path="loadEventsForDisabledDays" type="bool" default="false">
  Allows to load events for days that are not enabled.
  
  - If `true` - ignores `enabledDayPredicate` when calling `eventLoader`
  - If `false` - uses `enabledDayPredicate` to check when to call `eventLoader`
</ParamField>

### Predicate Functions

<ParamField path="enabledDayPredicate" type="bool Function(DateTime day)?">
  Function deciding whether given day should be enabled or not.
  
  If `false` is returned, this day will be disabled and use `disabledStyle`.
  
  ```dart
  enabledDayPredicate: (day) {
    // Disable weekends
    return day.weekday != DateTime.saturday && 
           day.weekday != DateTime.sunday;
  }
  ```
</ParamField>

<ParamField path="selectedDayPredicate" type="bool Function(DateTime day)?">
  Function deciding whether given day should be marked as selected.
  
  ```dart
  selectedDayPredicate: (day) {
    return isSameDay(_selectedDay, day);
  }
  ```
</ParamField>

<ParamField path="holidayPredicate" type="bool Function(DateTime day)?">
  Function deciding whether given day is treated as a holiday.
  
  Days marked as holidays will use `holidayStyle` from `CalendarStyle`.
  
  ```dart
  holidayPredicate: (day) {
    return holidays.contains(day);
  }
  ```
</ParamField>

## Callbacks

### Day Selection Callbacks

<ParamField path="onDaySelected" type="OnDaySelected?">
  Called whenever any day gets tapped.
  
  **Signature:**
  ```dart
  void Function(DateTime selectedDay, DateTime focusedDay)
  ```
  
  **Example:**
  ```dart
  onDaySelected: (selectedDay, focusedDay) {
    setState(() {
      _selectedDay = selectedDay;
      _focusedDay = focusedDay;
    });
  }
  ```
</ParamField>

<ParamField path="onDayLongPressed" type="OnDaySelected?">
  Called whenever any day gets long pressed.
  
  **Signature:**
  ```dart
  void Function(DateTime selectedDay, DateTime focusedDay)
  ```
</ParamField>

<ParamField path="onRangeSelected" type="OnRangeSelected?">
  Called whenever a day range gets selected.
  
  **Signature:**
  ```dart
  void Function(DateTime? start, DateTime? end, DateTime focusedDay)
  ```
  
  **Example:**
  ```dart
  onRangeSelected: (start, end, focusedDay) {
    setState(() {
      _rangeStart = start;
      _rangeEnd = end;
      _focusedDay = focusedDay;
    });
  }
  ```
</ParamField>

### Disabled Day Callbacks

<ParamField path="onDisabledDayTapped" type="void Function(DateTime day)?">
  Called whenever any disabled day gets tapped.
  
  This is triggered for days that are disabled by `enabledDayPredicate` or are outside the `firstDay`/`lastDay` range.
</ParamField>

<ParamField path="onDisabledDayLongPressed" type="void Function(DateTime day)?">
  Called whenever any disabled day gets long pressed.
</ParamField>

### Header Callbacks

<ParamField path="onHeaderTapped" type="void Function(DateTime focusedDay)?">
  Called whenever header gets tapped.
  
  Useful for implementing custom behavior like opening a date picker.
</ParamField>

<ParamField path="onHeaderLongPressed" type="void Function(DateTime focusedDay)?">
  Called whenever header gets long pressed.
</ParamField>

### Calendar State Callbacks

<ParamField path="onPageChanged" type="void Function(DateTime focusedDay)?">
  Called whenever currently visible calendar page is changed.
  
  This is triggered when the user swipes to a different month/week or when the calendar page changes programmatically.
  
  ```dart
  onPageChanged: (focusedDay) {
    _focusedDay = focusedDay;
  }
  ```
</ParamField>

<ParamField path="onFormatChanged" type="void Function(CalendarFormat format)?">
  Called whenever `calendarFormat` is changed.
  
  Required when using the format button or vertical swipe gestures to change formats.
  
  ```dart
  onFormatChanged: (format) {
    setState(() {
      _calendarFormat = format;
    });
  }
  ```
</ParamField>

<ParamField path="onCalendarCreated" type="void Function(PageController pageController)?">
  Called when the calendar is created. Exposes its PageController.
  
  Use this to programmatically control the calendar's page navigation.
</ParamField>

## Type Definitions

### OnDaySelected

Signature for `onDaySelected` callback. Contains the selected day and focused day.

```dart
typedef OnDaySelected = void Function(
  DateTime selectedDay,
  DateTime focusedDay,
);
```

### OnRangeSelected

Signature for `onRangeSelected` callback. Contains start and end of the selected range, as well as currently focused day.

```dart
typedef OnRangeSelected = void Function(
  DateTime? start,
  DateTime? end,
  DateTime focusedDay,
);
```

## Notes

<Note>
  The `TableCalendar` widget is generic, allowing you to specify your own event type. Use `TableCalendar<Event>` where `Event` is your custom event class.
</Note>

<Warning>
  Always provide `onFormatChanged` callback when using format buttons or vertical swipe gestures. Without this callback, format changes will have no effect.
</Warning>

<Note>
  All date-related properties (`firstDay`, `lastDay`, `focusedDay`, etc.) are automatically normalized to UTC format without their time component using the `normalizeDate` function.
</Note>

## Examples

### Basic Calendar

```dart
TableCalendar(
  firstDay: DateTime.utc(2010, 10, 16),
  lastDay: DateTime.utc(2030, 3, 14),
  focusedDay: _focusedDay,
  selectedDayPredicate: (day) {
    return isSameDay(_selectedDay, day);
  },
  onDaySelected: (selectedDay, focusedDay) {
    setState(() {
      _selectedDay = selectedDay;
      _focusedDay = focusedDay;
    });
  },
)
```

### Calendar with Events

```dart
TableCalendar<Event>(
  firstDay: DateTime.utc(2010, 10, 16),
  lastDay: DateTime.utc(2030, 3, 14),
  focusedDay: _focusedDay,
  eventLoader: (day) {
    return _getEventsForDay(day);
  },
  calendarStyle: CalendarStyle(
    markersMaxCount: 3,
  ),
)
```

### Range Selection Calendar

```dart
TableCalendar(
  firstDay: DateTime.utc(2010, 10, 16),
  lastDay: DateTime.utc(2030, 3, 14),
  focusedDay: _focusedDay,
  rangeStartDay: _rangeStart,
  rangeEndDay: _rangeEnd,
  rangeSelectionMode: RangeSelectionMode.toggledOn,
  onRangeSelected: (start, end, focusedDay) {
    setState(() {
      _rangeStart = start;
      _rangeEnd = end;
      _focusedDay = focusedDay;
    });
  },
)
```

### Custom Styled Calendar

```dart
TableCalendar(
  firstDay: DateTime.utc(2010, 10, 16),
  lastDay: DateTime.utc(2030, 3, 14),
  focusedDay: _focusedDay,
  calendarFormat: _calendarFormat,
  startingDayOfWeek: StartingDayOfWeek.monday,
  weekendDays: [DateTime.friday, DateTime.saturday],
  headerStyle: HeaderStyle(
    formatButtonVisible: true,
    titleCentered: true,
  ),
  calendarStyle: CalendarStyle(
    weekendTextStyle: TextStyle(color: Colors.red),
    outsideDaysVisible: false,
  ),
  onFormatChanged: (format) {
    setState(() {
      _calendarFormat = format;
    });
  },
)
```
